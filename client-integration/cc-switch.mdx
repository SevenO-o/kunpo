# CC Switch

[CC Switch](https://github.com/farion1231/cc-switch) 是开源跨平台桌面 GUI 工具，统一管理 Claude Code、Codex、Gemini CLI、OpenCode、OpenClaw 的 Provider 配置、API Key 切换、MCP 服务器与用量统计，支持 Windows / macOS / Linux。

配合 [KUNPO API](https://llm.ziy.cc) 使用后，你可以在多个 AI 编程工具之间快速切换，无需反复手动编辑 `~/.claude/settings.json` 或环境变量。

---

## 一、CC Switch 是什么？

CC Switch（项目地址：[github.com/farion1231/cc-switch](https://github.com/farion1231/cc-switch)）是由开发者 Jason Young（farion1231）维护的开源桌面应用，专为 AI 编程 CLI 设计的统一管理器。

如果你同时使用多个 AI 编程工具，可能会遇到这些问题：

- 每次切换 Provider，都要手动编辑配置文件或重新设置环境变量
- 多台设备之间配置无法同步，换机器就要从头配置
- WSL 与 Windows 各自维护一套配置，管理成本叠加

CC Switch 安装后常驻系统托盘，点击几下即可完成 AI 工具与服务商的切换。基于 Tauri 2 + Rust 构建，安装包轻巧，支持全平台。

### 支持的 CLI 工具

| 工具名称 | 开发方 | 简介 |
|----------|--------|------|
| Claude Code | Anthropic | Claude 模型驱动的 AI 编程助手，可在终端中对话、写代码、改 Bug |
| Codex | OpenAI | OpenAI 代码生成 CLI，支持多语言补全与生成 |
| Gemini CLI | Google | Google Gemini 模型的命令行版本 |
| OpenCode | 开源社区 | 开源 AI 编程 CLI，支持接入多种模型 |
| OpenClaw | 开源社区 | 兼容 OpenAI 接口的开源 CLI，可灵活切换上游 Provider |

## 二、核心功能

### Provider 管理

- 为每个 CLI 工具创建多个 Provider 配置
- 拖拽排序，常用配置置顶
- 一键切换当前激活的 Provider，无需重启 CLI
- 通过 Shared Config Snippet 切换服务商时保留插件与自定义设置

### MCP 服务器统一管理

在多个 CLI 中共用 MCP 服务器时，CC Switch 提供统一 MCP 管理面板，支持跨应用配置同步。

### 技能（Skills）安装

支持从 GitHub 仓库或 ZIP 文件安装技能包，通过 symlink 挂载，便于团队共享。

### 用量统计

内置使用量看板，可追踪每日/每周消费、请求次数、Token 用量与趋势图。

### 云同步

配置可同步至 Dropbox、OneDrive、iCloud（macOS）或 WebDAV 自建服务器。

## 三、安装

<Tabs>
  <Tab title="Windows">
    ```bash
    winget install --id=farion1231.CC-Switch -e
    ```

    或前往 [GitHub Releases](https://github.com/farion1231/cc-switch/releases) 下载 `.msi` / 便携版 `.zip`。系统要求：Windows 10+
  </Tab>

  <Tab title="macOS">
    ```bash
    brew tap farion1231/ccswitch
    brew install --cask cc-switch
    ```

    或下载 `.dmg` 拖入 Applications。系统要求：macOS 12+
  </Tab>

  <Tab title="Linux">
    ```bash
    # Debian / Ubuntu
    sudo dpkg -i cc-switch_x.x.x_amd64.deb
    # AppImage
    chmod +x cc-switch_x.x.x_amd64.AppImage && ./cc-switch_x.x.x_amd64.AppImage
    ```
  </Tab>
</Tabs>

### WSL 集成（Windows）

在 Windows 主机安装 CC Switch 后，可直接管理 WSL 内 Claude Code 等 CLI 的配置，无需在 WSL 内单独安装。

## 四、Claude Code 通过 CC Switch 接入 KUNPO API

### 为什么选择 KUNPO API？

KUNPO API 是统一的大模型网关，兼容 OpenAI 与 Anthropic 接口格式。通过一个 API Key 即可调用 Claude、GPT、Gemini、DeepSeek、GLM 等 150+ 模型，适合在 Claude Code 中灵活切换模型、控制成本。

### 连接信息

| 字段 | 值 |
|------|-----|
| 请求地址 | `https://llm.ziy.cc` |
| API Key | `sk-你的密钥`（在 [KUNPO 后台](https://llm.ziy.cc) 创建） |

### 推荐模型（Claude Code）

| 角色 | KUNPO 模型名 |
|------|----------------|
| 智谱 GLM（Claude Code 适配） | **`c/glm-5.2`** |
| Sonnet | `anthropic/claude-sonnet-4.6` |
| Opus | `anthropic/claude-opus-4.8` |
| Haiku | `anthropic/claude-haiku-4.5` |

<Tip>
  **`c/glm-5.2`** 已针对 Claude Code 完成专项适配，在工具调用、多轮对话与稳定性上优于通用 GLM 路由，强烈推荐在模型映射中使用。
</Tip>

> 模型 ID 必须与 [KUNPO API 模型广场](https://llm.ziy.cc) 完全一致。更多模型可在模型广场查询后填入模型映射表。

### 配置步骤

<Steps>
  <Step title="步骤 1：打开添加面板">
    打开 CC Switch 主界面，点击右上角橙色 **「+」** 按钮。

    <Frame caption="点击右上角 + 按钮">
      <img src="/images/cc-switch/1.png" alt="CC Switch 主界面，点击添加按钮" />
    </Frame>
  </Step>

  <Step title="步骤 2：选择自定义配置">
    在「添加新供应商」窗口中，确认位于「Claude 供应商」标签页，选择左上角 **「自定义配置」**。下滑配置参数。

    <Frame caption="选择自定义配置">
      <img src="/images/cc-switch/2.png" alt="Claude 供应商预设列表，选择自定义配置" />
    </Frame>
  </Step>

  <Step title="步骤 3：填写 API 连接信息">
    按以下信息填写表单：

    | 字段 | 值 |
    |------|-----|
    | 供应商名称 | `KUNPO API` |
    | 官网链接 | `https://llm.ziy.cc` |
    | API Key | `sk-你的密钥` |
    | 请求地址 | `https://llm.ziy.cc` |

    在 **「模型映射」** 区域，将各角色的「实际请求模型」填写为 KUNPO API 模型名：

    | 角色 | 实际请求模型 |
    |------|----------------|
    | 智谱 GLM（Claude Code 适配，推荐） | **`c/glm-5.2`** |
    | Sonnet | `anthropic/claude-sonnet-4.6` |
    | Opus | `anthropic/claude-opus-4.8` |
    | Haiku | `anthropic/claude-haiku-4.5` |

    <Note>
      请求地址填写 `https://llm.ziy.cc`，**不要以斜杠结尾**，**不要**加 `/v1`。若 CC Switch 提供「完整 URL」开关，保持关闭。
    </Note>

    <Frame caption="填写 API Key、请求地址与模型映射">
      <img src="/images/cc-switch/3.png" alt="填写 KUNPO API Key、请求地址和模型" />
    </Frame>

    确认信息无误后，点击 **「+ 添加」** 保存。
  </Step>

  <Step title="步骤 4：启用 Provider">
    返回主界面，在供应商列表中找到新添加的配置，点击 **「启用」** 按钮激活。

    <Frame caption="启用新添加的 Provider">
      <img src="/images/cc-switch/4.png" alt="启用 KUNPO API" />
    </Frame>
  </Step>

  <Step title="步骤 5：验证连接">
    在终端启动 Claude Code，输入 `/status` 查看配置是否生效：

    ```bash
    claude
    ```

    确认 **Anthropic base URL** 为 `https://llm.ziy.cc`，**Model** 为所配置的 KUNPO 模型名（如 `anthropic/claude-sonnet-4.6`）。

    也可使用 `/models` 切换其他已映射的模型。

    <Frame caption="在 Claude Code 中执行 /status 验证">
      <img src="/images/cc-switch/5.png" alt="Claude Code status 显示 KUNPO API 配置" />
    </Frame>
  </Step>
</Steps>

## 五、在 VSCode 中使用

Claude Code 提供官方 VSCode 扩展，与 CC Switch 完全兼容。

<Steps>
  <Step title="安装扩展">
    在 VSCode 扩展市场搜索 **Claude Code** 并安装。
  </Step>

  <Step title="配合 CC Switch 使用">
    在 CC Switch 中启用 KUNPO API Provider 后，VSCode 扩展会自动使用当前生效的配置，无需在 VSCode 内重复填写 API Key。右下角状态栏会显示 **Claude** 及 CC Switch 图标。
  </Step>

  <Step title="多工作区">
    多个 VSCode 窗口共享同一套 CC Switch 配置，切换一次全局生效。
  </Step>
</Steps>

## 六、其他 CLI 工具 + KUNPO API

Codex、Gemini CLI、OpenCode、OpenClaw 的配置方式与 Claude Code **类似**：在对应应用分区添加 Provider 时，选择 **「自定义配置」**，再填写 KUNPO 的连接信息（请求地址、API Key、模型名）。

### 通用连接信息

| 字段 | 值 |
|------|-----|
| 供应商名称 | `KUNPO API` |
| 官网链接 | `https://llm.ziy.cc` |
| API Key | `sk-你的密钥` |
| 请求地址 | `https://llm.ziy.cc` |

> 模型 ID 必须使用 [KUNPO API 模型广场](https://llm.ziy.cc) 中的名称。请求地址与 Claude Code 相同，填写 `https://llm.ziy.cc`，不要加 `/v1`。

### 通用配置步骤

1. 在 CC Switch 顶部切换到目标 CLI 分区（Codex / Gemini / OpenCode / OpenClaw）
2. 点击 **「+ 添加供应商」**
3. 选择 **「自定义配置」**，下滑进行参数配置
4. 填写上表中的 KUNPO 连接信息，并按该 CLI 要求填写模型名
5. 保存后返回主界面，确认新配置状态为「使用中」；若未启用，点击「启用」按钮激活

### Codex + KUNPO API

1. 按通用步骤在 **Codex** 分区选择 **「自定义配置」** 并填写 KUNPO 连接信息
2. 模型填写 [模型广场](https://llm.ziy.cc) 中的名称，如 `openai/gpt-5.5`
3. 启用后在终端运行 `codex`，使用 `/model` 切换模型

### Gemini CLI + KUNPO API

1. 安装 Gemini CLI：`npm install -g @google/gemini-cli`
2. 按通用步骤在 **Gemini** 分区选择 **「自定义配置」** 并填写 KUNPO 连接信息
3. 模型填写 `gemini-3.5-flash`（或模型广场中的其他 Gemini 模型名）
4. 启用后在终端运行 `gemini` 验证

### OpenCode + KUNPO API

1. 按通用步骤在 **OpenCode** 分区选择 **「自定义配置」** 并填写 KUNPO 连接信息
2. 模型填写 KUNPO 模型名，如 `openai/gpt-5.5`
3. 启用后在终端运行 `opencode` 验证

### OpenClaw + KUNPO API

1. 按通用步骤在 **OpenClaw** 分区选择 **「自定义配置」** 并填写 KUNPO 连接信息
2. 模型填写 KUNPO 模型名，如 `openai/gpt-5.5`
3. 启用后在终端运行 OpenClaw 验证

## 七、CC Switch vs Claude Code Router

| 对比维度 | CC Switch | Claude Code Router |
|----------|-----------|-------------------|
| 形态 | 桌面 GUI | CLI / 配置文件 |
| 支持的 CLI | 5 个 | 主要针对 Claude Code |
| Provider 预设 | 50+ | 依赖手动配置 |
| 本地代理 | 内置，支持热切换 | 需手动配置 |
| MCP 管理 | 统一界面 | 不支持 |
| WSL 支持 | 专项支持 | 有限 |
| 云同步 | 支持 | 不支持 |
| 用量统计 | 内置 Dashboard | 无 |
| 上手难度 | 低 | 中 |

## 八、常见问题

<AccordionGroup>
  <Accordion title="CC Switch 本身收费吗？">
    完全免费，MIT 开源。API 费用由 KUNPO API 按实际调用计费。
  </Accordion>

  <Accordion title="WSL 中如何让 CC Switch 生效？">
    CC Switch 安装在 Windows 侧，通过本地代理桥接 WSL 内请求。端口号可在 CC Switch 代理设置页查看。
  </Accordion>

  <Accordion title="切换 Provider 后需要重启 Claude Code 吗？">
    不需要。CC Switch 热切换在激活新 Provider 后立即生效。
  </Accordion>

  <Accordion title="配置数据存储在哪里？">
    本地数据库：`~/.cc-switch/cc-switch.db`；设置文件：`~/.cc-switch/settings.json`。
  </Accordion>

  <Accordion title="可以通过链接导入 Provider 配置吗？">
    可以。CC Switch 支持 Deep Link（`ccswitch://` 协议），团队可通过分享链接快速导入统一配置。
  </Accordion>

  <Accordion title="如何更换模型？">
    在 CC Switch 中编辑 KUNPO API Provider 的模型映射，或在 Claude Code 中使用 `/models` 切换。完整模型列表见 [KUNPO API 模型广场](https://llm.ziy.cc)。
  </Accordion>
</AccordionGroup>

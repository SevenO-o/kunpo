# API 总览

KUNPO API 基于统一网关 `https://llm.ziy.cc`，兼容 OpenAI 与 Anthropic 主流接口格式。一个 API Key 即可访问文本、图片、视频等多模态能力。

## 基础信息

| 项目 | 说明 |
|------|------|
| Base URL | `https://llm.ziy.cc` |
| 认证方式 | `Authorization: Bearer sk-你的密钥` |
| 文本协议 | OpenAI Chat Completions / Anthropic Messages |
| 图片协议 | OpenAI Images API（同步 + 异步） |
| 视频协议 | 异步任务（提交 + 轮询） |

<Note>
  尚未获取 API Key？请先阅读 [快速开始](/quick-start)。
</Note>

---

## 接口一览

| 能力 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 文本对话 | `POST` | `/v1/chat/completions` | OpenAI 兼容，支持流式、多模态 |
| Claude 原生 | `POST` | `/v1/messages` | Anthropic Messages API |
| 图片生成（同步） | `POST` | `/v1/images/generations` | 提交后等待完成，直接返回 URL |
| 图片生成（异步提交） | `POST` | `/v1/images/tasks` | 返回 `task_id`，适合长耗时模型 |
| 图片生成（异步查询） | `GET` | `/v1/images/tasks/:task_id` | 查询任务状态与结果 URL |
| 视频生成（提交） | `POST` | `/v1/video/generations` | 返回 `task_id` |
| 视频生成（查询） | `GET` | `/v1/video/generations/:task_id` | 查询任务状态与视频 URL |

## 协议格式对比

| 对比项 | Chat Completions | Messages API |
|--------|-----------------|--------------|
| 路径 | `/v1/chat/completions` | `/v1/messages` |
| 海外模型命名 | `anthropic/claude-sonnet-4.6` | `Claude-Sonnet-4.6` |
| 生态兼容 | OpenAI SDK 及兼容客户端 | Anthropic SDK / Claude Code |
| 适用场景 | 多模型切换、通用集成 | Claude 专属功能 |

<Tip>
  不确定用哪种格式？通用集成选 Chat Completions；Claude Code 或 Anthropic SDK 选 Messages API。
</Tip>

## 同步 vs 异步

部分能力提供两种调用模式：

| 模式 | 特点 | 适用接口 |
|------|------|----------|
| **同步** | 请求阻塞至任务完成，直接返回结果 | 文本对话、图片 GI/GI2/MI |
| **异步** | 提交后立即返回 `task_id`，客户端轮询结果 | 图片 Image-GPT2、豆包视频 |

异步任务通用流程：

```
1. POST 提交任务 → 获取 task_id
2. GET  轮询状态 → 直至 SUCCESS / completed
3. 读取 result_url / video_url
```

## 模型命名速查

| 类型 | 格式 | 示例 |
|------|------|------|
| 海外模型 | `供应商/模型名` | `google/gemini-3.1-flash-lite`、`openai/gpt-5.4` |
| 国内模型 | 直接使用名称 | `DeepSeek-R1-0528`、`Qwen3`、`GLM-4` |
| Claude Messages | 无前缀 `anthropic/` | `Claude-Sonnet-4.6` |
| DeepSeek Claude 兼容 | `c/` 前缀 | `c/deepseek-v4-pro` |

完整模型列表请访问 [KUNPO API 模型广场](https://llm.ziy.cc)。

## 常见 HTTP 状态码

| 状态码 | 含义 | 处理建议 |
|:------:|------|----------|
| `200` | 成功 | — |
| `400` | 请求参数错误 | 检查 JSON 格式与必填字段 |
| `401` | API Key 无效 | 检查 Authorization 头 |
| `429` | 请求频率超限 | 降低频率，稍后重试 |
| `503` | 模型无可用通道 | 确认模型名正确，或在控制台查看分组权限 |
| `502` / `504` | 网关超时 | 长耗时请求请耐心等待或改用异步接口 |

各接口的详细参数、示例与错误说明请参阅对应文档页。
